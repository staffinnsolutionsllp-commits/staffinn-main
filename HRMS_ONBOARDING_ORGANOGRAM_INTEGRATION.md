# HRMS Onboarding & Auto-Organogram Integration

## Overview
Employee onboarding ab automatically organogram create karega. Jab bhi koi naya employee onboard hoga, uska organogram node automatically ban jayega based on reporting structure.

## How It Works

### 1. Onboarding Process
When HR onboards a new employee through the 4-step form:

**Step 1: Basic Info & Contact**
- Personal details
- Contact information
- Emergency contacts

**Step 2: Identification & Banking**
- Aadhaar, PAN, UAN
- Bank account details

**Step 3: Employment & Organogram** (Critical Step)
- Department, Designation
- **Reporting Manager** (dropdown se select karo)
- **Role Level** (IC/Team Lead/Manager/Head)
- **Is People Manager?** (Yes/No)
- Team Name

**Step 4: Compensation & System Access**
- Salary details
- Role-based access
- Employee ID auto-generation

### 2. Automatic Organogram Creation

Jab employee onboard hota hai, system automatically:

1. **Employee Record Create** karta hai
2. **Organogram Node Create** karta hai with:
   - Employee ID link
   - Parent ID (reporting manager ka node)
   - Hierarchy Level (role level ke basis pe)
   - Position Title (designation)

**Level Mapping:**
```javascript
Head → Level 1
Manager → Level 2
Team Lead → Level 3
Individual Contributor → Level 4
```

### 3. Organogram Features

**View Modes:**
- **Diagram View**: Visual tree structure with connecting lines
- **List View**: Expandable/collapsible hierarchy list

**Capabilities:**
- ✅ Auto-generated from onboarding
- ✅ Edit any node (change reporting, position, employee)
- ✅ Delete nodes (children reassign to grandparent)
- ✅ Add manual positions (vacant positions)
- ✅ Real-time hierarchy visualization
- ✅ Shows filled vs vacant positions

**Statistics Dashboard:**
- Total Positions
- Filled Positions
- Vacant Positions

### 4. Editing Organogram

**Edit Node:**
- Click edit button on any node
- Change employee assignment
- Change reporting manager (parent)
- Update position title
- Modify hierarchy level

**Delete Node:**
- Click delete button
- Confirms before deletion
- Children automatically reassign to grandparent
- Maintains hierarchy integrity

**Add Manual Position:**
- Click "Add Position" button
- Select employee (or leave vacant)
- Choose parent node
- Set hierarchy level
- Enter position title

### Organization Chart Table
```javascript
{
  nodeId: "ORG001",
  employeeId: "EMP001", // Links to employee
  parentId: "ORG000", // Parent node (reporting manager's node)
  level: 2, // Hierarchy level
  position: "Senior Manager",
  children: ["ORG002", "ORG003"], // Child nodes
  createdAt: "2024-01-15T10:30:00Z"
}
```

## API Endpoints

### Onboarding
- `POST /api/hrms/candidates` - Create employee (auto-creates org node)

### Organogram
- `GET /api/hrms/organization/chart` - Get full hierarchy
- `POST /api/hrms/organization/nodes` - Create org node
- `PUT /api/hrms/organization/nodes/:id` - Update org node
- `DELETE /api/hrms/organization/nodes/:id` - Delete org node

## Workflow Integration

### Grievance Escalation
Organogram automatically defines escalation path:
```
Employee → Reporting Manager → Manager's Manager → HR → Management
```

### Approval Workflows
Leave/Claims approval follows organogram hierarchy:
```
Employee applies → Reporting Manager approves → HR processes
```

### Performance Reviews
ACR/Rating flows through reporting structure defined in organogram.

## Key Benefits

1. **Single Source of Truth**: Onboarding data automatically builds organogram
2. **No Manual Duplication**: HR doesn't need to create org structure separately
3. **Always Updated**: Any employee change reflects in organogram
4. **Workflow Ready**: Grievances, approvals, escalations work automatically
5. **Visual Clarity**: See company structure at a glance
6. **Easy Editing**: Reorganize structure with simple drag-and-edit

## Employee ID Strategy

**Auto-Generation:**
- Employee ID generated by HRMS on onboarding
- Format: EMP001, EMP002, etc.
- Unique across company

**Biometric Sync:**
- Same Employee ID synced to biometric device
- Attendance data references this ID
- Single ID across all systems

## Troubleshooting

**Issue: Organogram not updating after onboarding**
- Check if employee has reporting manager assigned
- Verify API response includes employeeId
- Check browser console for errors

**Issue: Circular reference error**
- Employee cannot report to themselves
- Check parent-child relationships
- Verify no loops in hierarchy

**Issue: Vacant positions showing**
- This is normal - positions can exist without employees
- Assign employee later through edit function
- Useful for planning future hires
